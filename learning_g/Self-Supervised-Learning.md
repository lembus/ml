# Self-Supervised Learning (SSL)

## 1. Motivation & Intuition

### The Problem: The "Label Bottleneck"
In traditional **Supervised Learning**, a machine learns by example. You show it a picture of a cat and explicitly tell it, "This is a cat." To train a robust model, you need millions of these labeled examples.
* **The Constraint:** Labeling data is expensive, slow, and hard to scale (e.g., hiring doctors to label X-rays or lawyers to classify legal clauses).
* **The Opportunity:** We have an infinite supply of *unlabeled* data (e.g., all text on the internet, all videos on YouTube).

### The Solution: Learning from the Data Itself
**Self-Supervised Learning (SSL)** allows a system to learn meaningful patterns without human labels. Instead of being told the answer, the model creates its own "puzzles" from the data and solves them.

> **Intuitive Analogy: The "Fill-in-the-Blank" Test** > Imagine you want to learn English, but you have no teacher to grade you. You pick up a book, cover a random word with your thumb, and try to guess what it is.  
> * *Sentence:* "The huge dog barked loudly at the _____."  
> * *Guess:* You might guess "postman," "cat," or "car." You likely *won't* guess "banana" or "quietly."  
> * *Verification:* You lift your thumb to see the real word.  
> 
> If you do this millions of times, you eventually learn grammar, context, and semantics—not because a teacher told you the rules, but because you forced yourself to predict missing parts of the data.

### Connection to Real-World Systems
In system design, SSL is used to create **Foundation Models**. Instead of training a new model from scratch for every specific task (like sentiment analysis or object detection), we pretrain a massive model using SSL on raw data. This model learns a "general understanding" of the domain. We then slightly tweak (fine-tune) this model for specific downstream tasks using very few labeled examples.

## 2. Conceptual Foundations

### Core Terminology
* **Pretext Task:** The synthetic problem the model solves to learn general features (e.g., "predict the missing word" or "rotate this image back to upright"). The task itself is discarded after training.
* **Encoder (Backbone):** The neural network architecture (e.g., ResNet, Vision Transformer) that converts raw input into a compact mathematical representation.
* **Representation (Embedding):** A vector that captures the semantic meaning of the input. In a well-trained representation space, vectors for "dogs" will cluster closely with "wolves," but remain distant from "cars."
* **Downstream Task:** The target real-world application (e.g., tumor classification), evaluated after the SSL pretraining phase.

### Major Paradigms

#### A. Pretext Tasks (Heuristic-Based)
* **Rotation Prediction:** Rotate an input image by 0°, 90°, 180°, or 270° and classify the applied rotation. Solving this requires the network to identify canonical object orientations and structural anatomy.
* **Jigsaw Puzzles:** Partition an image into a 3x3 grid, permute the tiles, and train the model to predict the original spatial permutation. This enforces learning of spatial contiguity and part-to-whole relationships.

#### B. Contrastive Learning (SimCLR, MoCo)
Instead of solving explicit spatial puzzles, the architecture learns to map semantically identical inputs to proximate metric spaces while repelling non-identical inputs.
* **Mechanism:** Generate two stochastic augmentations (views) of a single baseline anchor image.
* **Objective:** Maximize agreement between the two views of the same image (**Positive Pair**) while minimizing agreement between the anchor and all other batch instances (**Negative Pairs**).
* **SimCLR (Simple Framework for Contrastive Learning):** Relies on large computational batch sizes (e.g., 4096 instances) to supply a sufficiently diverse negative sample distribution within a single training step.
* **MoCo (Momentum Contrast):** Decouples batch size from negative sample capacity by maintaining a running memory queue of encoded negative keys.

#### C. Non-Contrastive Learning (BYOL)
* **BYOL (Bootstrap Your Own Latent):** Eliminates the requirement for negative samples entirely. The architecture uses an *Online Network* to predict the target representation generated by an asymmetric *Target Network*.
* **Key Insight:** Preventing representation collapse (where the network outputs a constant static vector for all inputs) is achieved through architectural asymmetry (adding a predictor MLP to the online branch) and applying a stop-gradient operator to the target branch.

#### D. Masked Modeling (BERT, MAE)
* **BERT (Text):** Stochastically mask 15% of input tokens and optimize for exact token reconstruction via cross-entropy.
* **MAE (Masked Autoencoders for Vision):** Mask a high proportion (e.g., 75%) of continuous image patches. The autoencoder must reconstruct the raw pixel values of the missing regions, forcing the encoder to model holistic visual semantics rather than local texture heuristics.

## 3. Mathematical Formulation

### Contrastive Loss (InfoNCE / NT-Xent)
Used in frameworks such as **SimCLR** and **MoCo**. 

Let $z_i$ and $z_j$ represent $L_2$-normalized embedding vectors originating from two augmented views of the exact same source instance. Let $\text{sim}(u, v) = u^T v$ denote the vector dot product (equivalent to cosine similarity for normalized vectors).

The loss objective for a specific positive pair $(i, j)$ across a total batch size of $2N$ augmented samples is formulated as:

$$
\mathcal{L}_{i,j} = -\log \frac{\exp(\text{sim}(z_i, z_j) / \tau)}{\sum_{k=1}^{2N} \mathbb{1}_{[k \neq i]} \exp(\text{sim}(z_i, z_k) / \tau)}
$$

* **Numerator:** $\exp(\text{sim}(z_i, z_j) / \tau)$ drives the positive pair representations $z_i$ and $z_j$ toward alignment.
* **Denominator:** The summation of similarities between anchor $z_i$ and all available batch instances $z_k$. Minimizing the total loss requires minimizing these negative dot products, pushing distinct instances apart in the latent space.
* **$\tau$ (Temperature Parameter):** A positive scalar regulating the sensitivity of the softmax distribution. Lower values amplify the gradient penalty assigned to "hard negatives" (negative samples situated suspiciously close to the anchor).

### Momentum Update Parameterization
In frameworks utilizing dual-network tracking (MoCo, BYOL), the Target Network weights $\theta_k$ do not receive direct gradient updates via backpropagation. Instead, they update via an exponential moving average (EMA) of the active Online Network parameters $\theta_q$:

$$
\theta_k \leftarrow m \theta_k + (1 - m) \theta_q
$$

* $m \in [0, 1)$ represents the momentum coefficient (typically initialized $\ge 0.999$).
* **Intuition:** This ensures the target projections evolve smoothly over training iterations, providing a consistent target distribution that prevents optimization divergence.

### BYOL Non-Contrastive Objective
BYOL minimizes the mean squared error between the $L_2$-normalized prediction of the online network and the target projection:

$$
\mathcal{L}_{\theta, \xi} = || \bar{q}_\theta(z_\theta) - \text{sg}(\bar{z}'_\xi) ||_2^2
$$

* $\bar{q}_\theta$: The predictor MLP network mapped exclusively to the online branch.
* $\text{sg}(\cdot)$: The stop-gradient operator enforcing zero parameter updates on the target branch $\xi$ during the backward pass.

## 4. Worked Examples

### End-to-End SimCLR Execution

#### Step 1: Sampling and Batch Augmentation
Assume a baseline mini-batch containing $N=2$ distinct raw images: Image A (Cat) and Image B (Dog). Applying stochastic transformations yields $2N=4$ total input views:
* $x_1$: Cat (Stochastically Cropped)
* $x_2$: Cat (Color Jittered) $\rightarrow$ **Positive Pair with $x_1$**
* $x_3$: Dog (Rotated)
* $x_4$: Dog (Grayscale) $\rightarrow$ **Positive Pair with $x_3$**

#### Step 2: Latent Encoding and Projection
All samples process sequentially through encoder $f(\cdot)$ and projection head $g(\cdot)$ to generate 2D unit-normalized vectors:
* $z_1$ (Cat Crop): $[0.90, 0.10]$
* $z_2$ (Cat Color): $[0.80, 0.60]$
* $z_3$ (Dog Rotate): $[-0.80, 0.60]$
* $z_4$ (Dog Grayscale): $[-0.90, 0.10]$

#### Step 3: Similarity Computation (Anchor $z_1$)
Compute pairwise vector dot products relative to anchor $z_1$:
* **Positive Match ($z_1 \cdot z_2$):** $(0.90 \times 0.80) + (0.10 \times 0.60) = 0.78$
* **Negative Match 1 ($z_1 \cdot z_3$):** $(0.90 \times -0.80) + (0.10 \times 0.60) = -0.66$
* **Negative Match 2 ($z_1 \cdot z_4$):** $(0.90 \times -0.90) + (0.10 \times 0.10) = -0.80$

#### Step 4: Loss Optimization
Evaluating the NT-Xent loss (assuming $\tau=1.0$):

$$
\mathcal{L}_{1,2} = -\log \frac{e^{0.78}}{e^{0.78} + e^{-0.66} + e^{-0.80}} = -\log \frac{2.181}{2.181 + 0.517 + 0.449} = -\log(0.693) \approx 0.367
$$

Because the positive similarity ($0.78$) is significantly higher than the negative similarities ($-0.66, -0.80$), the model incurs a low scalar loss. If the network erroneously projected $z_3$ to $[0.85, 0.15]$ (similarity $\approx 0.78$), the denominator would expand, driving the loss upward and generating strong backpropagation gradients to separate the cat and dog embeddings.

## 5. Relevance to Machine Learning Practice

### Applied Production Domains
1. **Natural Language Processing:** Large Language Models (BERT, GPT series, LLaMA) rely entirely on self-supervised objectives (Masked Token Reconstruction or Autoregressive Next-Token Prediction) over internet-scale text corpora.
2. **Computer Vision:** Pretraining Vision Transformers (ViTs) or deep residual networks on massive uncurated datasets (e.g., JFT-300M) before transferring weights to specialized downstream tasks such as pathology slide segmentation or satellite imagery analysis.
3. **Sequential Recommendation Systems:** Modeling historical user interaction logs (e.g., BERT4Rec) to predict subsequent behavioral engagements without requiring explicit user rating feedback.

### Architectural Trade-Offs

| Dimension | Characteristic Impact |
| :--- | :--- |
| **Computational Overhead** | Extreme pretraining infrastructure demands (multi-node GPU clusters required for convergence). |
| **Feature Robustness** | Highly resilient to downstream label noise and class imbalance compared to supervised baselines. |
| **Inductive Bias Propagation** | Models absorb upstream dataset biases verbatim; uncurated scraping introduces demographic and structural skews into feature representations. |
| **Evaluation Complexity** | Pretext validation metrics do not correlate linearly with downstream task accuracy. |

### Deployment Decision Matrix
* **Implement SSL when:** Unlabeled domain data exceeds labeled instances by orders of magnitude (e.g., >1M raw samples vs. <10k labeled instances).
* **Bypass SSL when:** The target domain is structurally simple, fully labeled, or operates under strict latency/compute budgets where transfer learning from standard supervised weights achieves parity.

## 6. Common Pitfalls & Misconceptions

### 1. Latent Representation Collapse
* **Failure Mode:** The optimization converges on a degenerate trivial solution where the encoder outputs an identical constant vector for all inputs ($z_i \equiv c$). Under this condition, positive similarity equals 1.0 and loss drops to zero without learning meaningful features.
* **Remediation:** Ensure adequate negative sampling distributions (Contrastive architectures) or strictly enforce architectural asymmetry alongside target stop-gradients (Predictor-based architectures).

### 2. Augmentation Shortcut Learning
* **Failure Mode:** The network exploits low-level physical artifacts to solve the pretext task rather than learning semantic representations.
* **Example:** In rotation prediction tasks, networks may track lens chromatic aberration or border padding irregularities to identify image orientation. In contrastive vision setups, omitting color jitter allows models to match positive pairs strictly via global color histograms.

### 3. Misinterpreting Pretext Convergence
* **Misconception:** Assuming that a steadily decreasing SSL training loss guarantees improved generalization on target downstream benchmarks.
* **Reality:** Models frequently overfit to the mechanics of the synthetic pretext task. Validation must be conducted concurrently via **Linear Probing** (evaluating a linear classifier trained over frozen encoder weights on a standardized validation set).

## Interview Preparation Section

### Foundational Questions

**Q1: Clearly distinguish between Supervised, Unsupervised, and Self-Supervised learning paradigms.**
* **Answer:** * **Supervised:** Operates on paired datasets $(X, Y)$. The objective minimizes prediction error against explicit human-curated labels $Y$.
  * **Unsupervised:** Operates strictly on unlabeled distributions $X$. The objective identifies intrinsic topological structures (e.g., density estimation, clustering).
  * **Self-Supervised:** Operates on unlabeled distributions $X$, but autonomously constructs deterministic pseudo-labels $Y'$ from the data structural properties. It leverages the optimization stability of supervised loss formulations to solve unsupervised representation problems.

**Q2: What is the architectural justification for appending a non-linear projection head $g(\cdot)$ during contrastive pretraining, only to discard it during downstream fine-tuning?**
* **Answer:** The projection head maps internal encoder representations into a metric space optimized explicitly for the contrastive loss objective. To minimize contrastive loss, this final space must become strictly invariant to applied augmentations (e.g., color transformations, spatial cropping). However, certain downstream tasks rely on these exact discarded properties (e.g., color distribution is critical for fine-grained species classification). By forcing the invariance filtering to occur within the downstream projection MLP $g(\cdot)$, the upstream representation inside the core encoder $f(\cdot)$ retains richer, multi-purpose structural features.

---

### Mathematical Questions

**Q3: Analyze the boundary behavior of the NT-Xent loss function as the temperature hyperparameter $\tau$ approaches its limits ($\tau \to 0$ and $\tau \to \infty$).**
* **Answer:**
  * **As $\tau \to 0$:** The exponential weighting inside the softmax distribution sharpens exponentially, effectively transitioning into an exact argmax operator. The loss calculation becomes dominated entirely by the single hardest negative sample in the batch. Gradients derived from moderately distant negatives vanish to zero. This induces severe optimization instability and jagged gradient trajectories.
  * **As $\tau \to \infty$:** The softmax distribution flattens into a uniform probability distribution. The spatial proximity of negative embeddings relative to the anchor becomes irrelevant; all negative samples exert identical repulsive force regardless of metric distance. The network loses the capacity to refine fine-grained semantic boundaries.

**Q4: Prove or explain mathematically why the BYOL architecture avoids representation collapse despite omitting negative repulsive forces.**
* **Answer:** Let the online network output be $q_\theta(z_\theta)$ and the target output be $z'_\xi$. In a standard symmetric architecture without stop-gradients, the gradient of the MSE loss $\mathcal{L} = ||q_\theta(z_\theta) - z'_\xi||_2^2$ with respect to $\xi$ would directly update the target weights to match the online predictions. This creates an immediate mathematical shortcut: $\theta \to c, \xi \to c \implies \mathcal{L} = 0$. 

  By introducing the stop-gradient operator $\text{sg}(\cdot)$, the optimization step modifies parameters $\theta$ exclusively:
  
  $$\nabla_\theta \mathcal{L} = 2 (q_\theta(z_\theta) - z'_\xi) \nabla_\theta q_\theta(z_\theta)$$

  The target branch parameters $\xi$ update strictly via the momentum trajectory $\xi \leftarrow m\xi + (1-m)\theta$. Consequently, the target network acts as a continuous tracking estimator of historical online states. The optimization tracks a dynamic moving target rather than a static equilibrium, forcing the online predictor to perform an ongoing variance-maximization projection similar to implicit Principal Component Analysis.

### Applied & System Design Questions

**Q5: You are tasked with designing an automated visual inspection pipeline for semiconductor manufacturing. You possess 2 million uncurated wafer imagery captures, but budget constraints allow for only 400 expert-labeled defect annotations. Detail your end-to-end system architecture.**
* **Answer:**
  1. **Pretraining Phase:** Deploy a Masked Autoencoder (MAE) or SimCLR backbone over the 2 million unlabeled images. Augmentation pipelines must be domain-restricted: spatial cropping and arbitrary 90° rotations are structurally valid for wafer topography, whereas aggressive color jittering or Gaussian blurring must be excluded to preserve microscopic defect visibility.
  2. **Feature Extraction:** Freeze the optimized encoder weights $f(\cdot)$. Pass all 400 labeled samples through the backbone to extract uniform representation vectors.
  3. **Downstream Modeling:** Train a low-capacity classifier (e.g., regularized Logistic Regression or Support Vector Machine) directly on the 400 extracted vector embeddings. Low model capacity prevents overfitting on the scarce label set.
  4. **Fallback Anomaly Detection:** In parallel, fit a Gaussian Mixture Model or One-Class SVM over the unlabeled embedding distribution to flag structural outliers directly at inference time.

**Q6: Account for the disparate masking ratio heuristics adopted by vision modeling (MAE masks 75% of patches) versus language modeling (BERT masks 15% of tokens).**
* **Answer:** The divergence stems from intrinsic differences in **Information Density** across modalities:
  * **Visual Signals:** Raw pixel lattices exhibit high spatial redundancy. If an image is masked at a 15% ratio, deep networks can reconstruct missing patches trivially by applying low-level mathematical interpolation from immediately adjacent pixels without learning high-level object semantics. Forcing a 75% masking threshold removes local continuity, compelling the network to infer holistic structural geometry (e.g., reconstructing a missing vehicle chassis based solely on a visible wheel and window).
  * **Linguistic Signals:** Human language is information-dense and highly non-linear. Every individual token carries concentrated semantic weight. Masking 75% of a textual sequence destroys sentence syntax and contextual framing entirely, rendering the reconstruction task mathematically ill-posed. A 15% threshold provides sufficient contextual anchors to deduce missing semantic concepts.

### Debugging & Failure Modes

**Q7: During contrastive pretraining on a custom dataset, your training loss decreases steadily toward zero, yet downstream linear probing accuracy remains near random chance. Identify primary root causes.**
* **Answer:**
  1. **Destructive Augmentation:** The transformation pipeline is systematically stripping invariant features required for downstream discrimination. For example, applying aggressive random cropping to a medical classification task where pathology is localized to a tiny structural margin.
  2. **Insufficient Batch Sampling:** If training SimCLR on restricted hardware with small batch sizes (e.g., $N=32$), the negative sample pool lacks semantic diversity. The network easily pushes apart distinct low-level textures without ever encountering hard semantic negatives.
  3. **Distribution Shift in Preprocessing:** The normalization constants (mean/standard deviation) or interpolation algorithms used during downstream evaluation deviate from those enforced during the SSL pretraining runs.

**Q8: While training a BYOL architecture, network tracking loss begins oscillating violently before diverging to NaN values. What architectural diagnostic steps do you execute?**
* **Answer:**
  1. **Batch Normalization Synchronization:** Verify the placement of Batch Normalization layers within the predictor MLP. BYOL relies critically on BN across intermediate linear projections to maintain representation variance. Ensure layer statistics are properly synchronized across distributed GPU processes.
  2. **Momentum Annealing Schedule:** Inspect the momentum parameter $m$. If initialized too low (e.g., $m=0.90$), the target network parameters evolve too rapidly, preventing the online predictor branch from tracking the shifting target space. Ensure $m$ follows a cosine annealing schedule starting at $\ge 0.996$ and terminating at $1.0$.
  3. **Learning Rate Scaling:** Check for gradient explosion within the un-normalized predictor head; apply strict gradient clipping ($\le 1.0$) or reduce the base learning rate of the online optimizer.

## 9. Modern SSL Paradigms: Self-Distillation and Latent Prediction

### A. Self-Distillation (DINO / DINOv2)

#### Motivation
Both contrastive methods (SimCLR) and non-contrastive methods (BYOL) require specific architectural choices to prevent representational collapse — large batches and hard negatives for SimCLR, or architectural asymmetry and stop-gradients for BYOL. **DINO** introduces a cleaner collapse-prevention mechanism based on knowledge distillation between a student and a teacher — where the teacher is the student's own historical self, outputs are normalized via a centering operation, and no negative pairs are ever needed.

DINOv2 extends DINO to produce features with remarkable emergent properties (explicit depth estimation and semantic segmentation emerge without supervision), enabled by curated training data and a combined self-distillation + masked prediction objective.

#### Conceptual Foundations

**Two-Crop Strategy:**
* A high-resolution "global" crop ($224 \times 224$) captures the full scene.
* Multiple low-resolution "local" crops ($96 \times 96$) capture specific object parts.
* The **student network** $g_{\theta_s}$ processes both global and local crops.
* The **teacher network** $g_{\theta_t}$ (an EMA of the student, $\theta_t \leftarrow m \theta_t + (1-m) \theta_s$) processes only the global crops.
* The student is trained to match the teacher's output distribution — local crops must predict what the teacher sees globally, forcing multi-scale semantic consistency.

**Centering:**
The teacher's output logits are centered by subtracting a running mean vector $c$, updated as an EMA of past teacher batch outputs. Without centering, the teacher collapses to outputting maximum probability on a single prototype for all inputs (a trivially minimal cross-entropy solution). Centering forces the output distribution to spread across all prototype dimensions.

**DINOv2 Extensions Over DINO:**
* Adds a SwAV-style online clustering regularization term that distributes representations uniformly across prototype codes.
* Adds a masked image modeling term (iBOT) providing patch-level supervision alongside the global self-distillation objective.
* Uses a curated training dataset (LVD-142M) with deduplication and quality filtering, rather than raw internet scrapes.
* Scales to ViT-g (1.1B parameters) — DINOv2 features linearly predict depth maps, semantic segmentation, and object part correspondences without any task-specific fine-tuning, properties absent from CLIP and supervised ImageNet models.

#### Mathematical Formulation

Let the student and teacher outputs be projected through separate softmax functions over $K$ prototype dimensions with temperatures $\tau_s > \tau_t$ (teacher is sharpened for peakier targets):

$$P_s^{(i)}(x) = \frac{\exp(g_{\theta_s}(x)^{(i)} / \tau_s)}{\sum_k \exp(g_{\theta_s}(x)^{(k)} / \tau_s)}$$

$$P_t^{(i)}(x) = \frac{\exp\left((g_{\theta_t}(x)^{(i)} - c^{(i)}) / \tau_t\right)}{\sum_k \exp\left((g_{\theta_t}(x)^{(k)} - c^{(k)}) / \tau_t\right)}$$

where the centering vector $c$ is updated each step: $c \leftarrow m_c \cdot c + (1 - m_c) \cdot \text{BatchMean}(g_{\theta_t}(x))$.

For an image with one global view $x_g$ and local views $\{x_l^{(j)}\}$, the total loss minimizes student-teacher cross-entropy:

$$\mathcal{L}_{\text{DINO}} = -\sum_j H\!\left(P_t(x_g),\, P_s(x_l^{(j)})\right) - H\!\left(P_t(x_g),\, P_s(x_g)\right)$$

where $H(a, b) = -\sum_k a^{(k)} \log b^{(k)}$ is cross-entropy. The sharpened teacher distribution provides a crisp learning signal; the centered teacher prevents degenerate solutions; the asymmetric crop setup (student sees local, teacher sees global) enforces coarse-to-fine semantic consistency.

### B. Joint-Embedding Predictive Architectures (I-JEPA)

#### Motivation
Masked autoencoders (MAE) predict raw pixel values for masked patches. This objective is dominated by low-level solutions: a model can achieve near-zero pixel MSE by exploiting local texture periodicity or blurring from adjacent patches — without learning high-level semantic content. The reconstruction target is fundamentally too low-level.

**I-JEPA (Image Joint-Embedding Predictive Architecture)** implements a key insight: **predict in representation space, not in pixel space**. Instead of predicting exact pixel intensities, the model predicts the abstract representation of masked regions as produced by a teacher encoder. Because the teacher has already compressed patches into semantic abstractions (discarding exact pixel values), the predictor is forced to infer semantic content rather than texture statistics.

#### Conceptual Foundations

**Four Components:**

1. **Context Encoder** $f_\theta$: Processes the visible (unmasked) patches. Produces context representations.
2. **Predictor** $g_\phi$: A lightweight Transformer that takes context representations + positional queries for target block locations. Outputs predicted representations for each target block.
3. **Target Encoder** $h_\xi$: An EMA copy of the context encoder ($\xi \leftarrow m\xi + (1-m)\theta$). Receives the full image (all patches) and produces target representations. No gradient flows through this branch.
4. **Masking Strategy:** Rather than masking random individual patches, I-JEPA masks contiguous spatial blocks (e.g., 4 rectangular blocks averaging $25\%$ coverage each), making the prediction task semantically harder — the model must infer an entire object region, not just fill in a texture gap.

**Why This Works Without Collapse:**
The stop-gradient on the target encoder and the EMA update rule (identical to BYOL's target network) prevent the trivial solution where both encoder and predictor output constants — the predictor must continuously track a slowly evolving target rather than a fixed equilibrium.

#### Mathematical Formulation

Let $\mathcal{C}$ be the set of unmasked context patch indices and $\{B_1, \ldots, B_K\}$ be the $K$ target block regions. The context encoder produces:

$$Z_{\mathcal{C}} = f_\theta\!\left(x_{\mathcal{C}}\right)$$

For each target block $B_k$, the predictor generates a predicted representation using context embeddings and positional queries for the target positions:

$$\hat{z}_{B_k} = g_\phi\!\left(Z_{\mathcal{C}},\; \text{pos}(B_k)\right)$$

The target encoder provides ground-truth representations:

$$z_{B_k} = h_\xi\!\left(x_{B_k}\right)$$

Training objective (L2 loss in representation space, averaged over all target blocks):

$$\mathcal{L}_{\text{I-JEPA}} = \frac{1}{K} \sum_{k=1}^{K} \left\lVert \hat{z}_{B_k} - \text{sg}(z_{B_k}) \right\rVert_2^2$$

where $\text{sg}(\cdot)$ denotes the stop-gradient operator.

#### Comparison: MAE vs. I-JEPA

| Dimension | MAE | I-JEPA |
| :--- | :--- | :--- |
| **Prediction Target** | Raw pixel values | Abstract representations |
| **Mask Strategy** | Random individual patches | Contiguous spatial blocks |
| **Training Efficiency** | Requires many epochs ($\sim$1600) | Converges faster ($\sim$600 epochs) |
| **Linear Probe Accuracy** | Moderate (reconstructing pixels ≠ learning semantics) | Higher (semantic representations emerge naturally) |
| **Decoder Required** | Yes (pixel decoder) | No (predictor is lightweight) |

### C. Multi-Modal SSL: Where CLIP Fits

CLIP demonstrates that SSL can operate across modalities using **naturally co-occurring paired data** (images + alt-text), bypassing the need to construct synthetic pretext tasks entirely.

**Natural Supervision at Scale:** Rather than designing a pretext task, CLIP mines 400M naturally aligned image-text pairs from the internet. The implicit pretext task is: correctly match each image to its caption within a large batch. This is structurally supervised (labels exist — the pairings), but labels are freely available from web scraping, requiring no human annotation.

**Formal Connection to Contrastive SSL:**
CLIP's InfoNCE objective is mathematically identical to SimCLR's NT-Xent loss:
* SimCLR defines positive pairs as two augmented views of the **same image**.
* CLIP defines positive pairs as an image and its **natural language description**.

The substitution of the augmented-view positive pair with a cross-modal positive pair allows the shared embedding space to align visual and linguistic semantic concepts. The model learns that `[image of a dog]` and `"a golden retriever playing fetch"` should be geometrically proximate, without ever receiving an explicit "dog" class label.

**Data2Vec (Universal SSL):** Meta's Data2Vec applies a single unified SSL objective across modalities using a single architecture:
1. A **teacher** (EMA of student) processes the full unmasked input and outputs top-$K$ layer representations.
2. A **student** processes the masked input and predicts the teacher's averaged representations at masked positions.
The only modality-specific components are the input tokenizer (patch extractor for images, subword tokenizer for text, waveform CNN for audio) and the masking strategy. The prediction target and loss are identical across all modalities — demonstrating that semantic representation learning has a common computational structure regardless of input type.

## 10. Extended Interview Questions

### DINO / DINOv2

**Q9: Explain why centering is critical in DINO but not necessary in BYOL.**
**Answer:** In DINO, the teacher outputs a softmax probability distribution over $K$ discrete prototype slots. Without centering, the trivially optimal cross-entropy solution is to collapse all probability mass onto a single prototype for every input (constant distribution across the dataset). The loss reaches zero — not because the representations are meaningful, but because both student and teacher have learned to output the same degenerate constant. Centering subtracts the running batch mean from teacher logits, ensuring the output distribution cannot concentrate entirely on one slot and forcing the network to use all prototype dimensions. BYOL avoids representational collapse via a different mechanism: the predictor MLP on the student branch must project to match the teacher's output, but since the teacher (a lagged EMA) is always slightly ahead of the student, the student must continuously change its representations. There is no fixed equilibrium to collapse to. This structural asymmetry (predictor vs. stop-gradient) replaces the role that centering plays in DINO.

**Q10: DINOv2 features encode geometric properties (depth, segmentation) that supervised ImageNet features do not. What aspect of the SSL objective enables this geometric structure to emerge?**
**Answer:** Supervised ImageNet training optimizes for 1,000 discrete object category classifications. Gradient pressure systematically prunes any feature dimensions that do not contribute to category separation — including scene geometry, spatial layout, and structural continuity, which are not discriminative between ImageNet categories (all cat images need similar features regardless of depth or pose). DINOv2's combined objective (self-distillation + masked patch prediction) does not specify which visual properties are valuable. The masked prediction term forces the encoder to preserve information about spatial structure and object boundaries in order to predict the representations of masked image regions — because the representation of a masked patch depends heavily on its geometric relationship to visible patches. Geometric features emerge as a necessary byproduct of the spatial prediction requirement, not because they were explicitly supervised.

### I-JEPA

**Q11: I-JEPA predicts in representation space rather than pixel space. Explain concretely why predicting pixels is a fundamentally easier but less useful task.**
**Answer:** Predicting raw pixel values for masked patches can be accomplished through multiple degenerate low-level strategies without learning semantic content: (1) predicting the global image mean color achieves moderate pixel MSE with zero semantic understanding; (2) blurring from immediately adjacent unmasked pixels (bilinear interpolation) reconstructs smooth textures with low MSE in homogeneous regions; (3) exploiting texture periodicity (grass, fabric, sky) produces accurate pixel predictions through memorized pattern repetition. All three strategies work purely at the level of local statistics and never require understanding what object is depicted or why specific pixels have specific values. In representation space, the target encoder has already executed semantic compression — discarding exact pixel intensities and retaining high-level features (object categories, part identities, scene attributes). The predictor cannot interpolate from adjacent textures in this space; it must infer the abstract semantic content of the masked region. This forces the model to learn genuine visual concepts rather than texture statistics.

### Multi-Modal SSL

**Q12: CLIP uses naturally co-occurring image-text pairs rather than a synthetic pretext task. What are the failure modes introduced by relying on web-scraped alt-text for supervision?**
**Answer:** Web alt-text introduces three systematic biases: (1) **Descriptive sparsity** — many alt-text strings are non-descriptive metadata (`"IMG_4821.jpg"`, `"Click here"`) rather than semantic image descriptions. The model learns to align visual features with noise, degrading precision in the semantic embedding space. (2) **Demographic and cultural bias** — alt-text reflects the cultural and linguistic biases of its authors. If a concept (e.g., "wedding") is predominantly described in the context of specific cultural practices in the training data, the image embedding for all wedding images will cluster in a culturally biased region of the semantic space, producing discriminatory zero-shot classifications. (3) **Spatial relationship blindness** — alt-text almost never describes the relative spatial arrangement of objects ("the dog is to the left of the cat"). CLIP therefore learns strong object recognition but systematically fails at spatial reasoning tasks (compositional VQA, grounding). These failure modes are inherited by any downstream system built on CLIP features.
